================================================================================
SIR PINNs PROJECT — LINE-BY-LINE CODE EXPLANATION
================================================================================

This document explains every line in each file of the SIR Physics-Informed
Neural Network (PINN) project. The project learns solutions to the SIR epidemic
model by training a neural network to satisfy the governing differential
equations, initial conditions, and population conservation.

Line 4: def sir_residuals(t, model, beta, gamma):
--------------------------------------------------------------------------------
Defines a function that computes how far the network's predictions violate the
SIR differential equations. beta is infection rate; gamma is recovery rate.

--------------------------------------------------------------------------------
Line 5-11: (docstring)
--------------------------------------------------------------------------------
Multi-line docstring documenting the three ODEs the residuals enforce:
  dS/dt = -beta * S * I
  dI/dt = beta * S * I - gamma * I
  dR/dt = gamma * I

--------------------------------------------------------------------------------
Line 12:     t = t.requires_grad_(True)
--------------------------------------------------------------------------------
Marks tensor t as requiring gradients so PyTorch can compute dS/dt, dI/dt, and
dR/dt via automatic differentiation. requires_grad_(True) modifies t in place.

--------------------------------------------------------------------------------
Line 13:     output = model(t)
--------------------------------------------------------------------------------
Runs the neural network on collocation time points t to get predicted S, I, R.

--------------------------------------------------------------------------------
Line 14:     s, i, r = output[:, 0:1], output[:, 1:2], output[:, 2:3]
--------------------------------------------------------------------------------
Splits the 3-column output into separate tensors for S, I, and R. Slicing
0:1 keeps shape (N, 1) instead of (N,) so autograd.grad works consistently.

--------------------------------------------------------------------------------
Line 15: (blank line)
--------------------------------------------------------------------------------
Blank line for readability.

--------------------------------------------------------------------------------
Line 16:     ds_dt = torch.autograd.grad(s, t, grad_outputs=torch.ones_like(s), create_graph=True)[0]
--------------------------------------------------------------------------------
Computes derivative of s with respect to t (dS/dt) at each collocation point.
grad_outputs=torch.ones_like(s) scales the backward pass. create_graph=True
allows second-order derivatives during training. [0] extracts the gradient
tensor from the returned tuple.

--------------------------------------------------------------------------------
Line 17:     di_dt = torch.autograd.grad(i, t, grad_outputs=torch.ones_like(i), create_graph=True)[0]
--------------------------------------------------------------------------------
Same as line 16, but for dI/dt.

--------------------------------------------------------------------------------
Line 18:     dr_dt = torch.autograd.grad(r, t, grad_outputs=torch.ones_like(r), create_graph=True)[0]
--------------------------------------------------------------------------------
Same as line 16, but for dR/dt.

--------------------------------------------------------------------------------
Line 19: (blank line)
--------------------------------------------------------------------------------
Blank line separating derivative computation from residual formulas.

--------------------------------------------------------------------------------
Line 20:     f_s = ds_dt + beta * s * i
--------------------------------------------------------------------------------
Residual for the S equation. Should be zero if dS/dt = -beta*S*I. Nonzero
values mean the network violates the susceptible ODE.

--------------------------------------------------------------------------------
Line 21:     f_i = di_dt - beta * s * i + gamma * i
--------------------------------------------------------------------------------
Residual for the I equation. Should be zero if dI/dt = beta*S*I - gamma*I.

--------------------------------------------------------------------------------
Line 22:     f_r = dr_dt - gamma * i
--------------------------------------------------------------------------------
Residual for the R equation. Should be zero if dR/dt = gamma*I.

--------------------------------------------------------------------------------
Line 23: (blank line)
--------------------------------------------------------------------------------
Blank line before return statement.

--------------------------------------------------------------------------------
Line 24:     return f_s, f_i, f_r
--------------------------------------------------------------------------------
Returns the three residual tensors for use in the physics loss (squared and

--------------------------------------------------------------------------------
Defines loss for matching initial conditions at t = 0. t0 is time zero; s0, i0,
r0 are target initial values for S, I, R.

--------------------------------------------------------------------------------
Line 28:     """Penalty for mismatch with initial conditions at t = 0."""
--------------------------------------------------------------------------------
Docstring explaining this loss penalizes deviation from known starting values.

--------------------------------------------------------------------------------
Line 29:     output = model(t0)
--------------------------------------------------------------------------------
Evaluates the network at t = 0 to get predicted initial S, I, R.

--------------------------------------------------------------------------------
Line 30:     s, i, r = output[:, 0], output[:, 1], output[:, 2]
--------------------------------------------------------------------------------
Extracts the three output columns as 1D tensors (only one time point at t0).

--------------------------------------------------------------------------------
Line 31:     return (
--------------------------------------------------------------------------------
Starts a multi-line return expression for the total initial-condition loss.

--------------------------------------------------------------------------------
Line 32:         torch.mean((s - s0) ** 2)
--------------------------------------------------------------------------------
Mean squared error between predicted S at t=0 and target s0.

--------------------------------------------------------------------------------
Line 33:         + torch.mean((i - i0) ** 2)
--------------------------------------------------------------------------------
Mean squared error for I at t=0 vs i0; added to the S term.

--------------------------------------------------------------------------------
Line 34:         + torch.mean((r - r0) ** 2)
--------------------------------------------------------------------------------
Mean squared error for R at t=0 vs r0; added to S and I terms.

--------------------------------------------------------------------------------
Line 35:     )
--------------------------------------------------------------------------------
Closes the return expression; sum of three MSE terms is the IC loss.

Line 36: (blank line)
--------------------------------------------------------------------------------
Blank line separating functions.

Line 37: (blank line)
--------------------------------------------------------------------------------
Blank line for readability.

--------------------------------------------------------------------------------
Line 38: def conservation_loss(model, t):
--------------------------------------------------------------------------------
Defines loss enforcing S + I + R = 1 (normalized total population).

--------------------------------------------------------------------------------
Line 39:     """Penalty when S + I + R deviates from 1."""
--------------------------------------------------------------------------------
Docstring: populations are fractions of total population, so their sum must be 1.

--------------------------------------------------------------------------------
Line 40:     output = model(t)
--------------------------------------------------------------------------------
Network predictions at all time points in t.

--------------------------------------------------------------------------------
Line 41:     return torch.mean((output.sum(dim=1) - 1.0) ** 2)
--------------------------------------------------------------------------------
Sums S+I+R along dim=1 (per row/time point), subtracts 1, squares the error,
and averages. Penalizes any violation of population conservation.
